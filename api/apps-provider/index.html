Applications and Application Providers

Interface: IApplication

application.name

An application object must have a String property called name that is equal to the name of the application.

application.callMethod(connection, interfaceName, methodName, args)

  • connection {Connection} connection that received a call packet.
  • interfaceName {String} name of an interface.
  • methodName {String} name of a method.
  • args {Array} array of method arguments (there is always at least one item in the array and the last one is always a callback).

Calls a method of the application.

If the specified interface (or method) does not exist, then callMethod should invoke the callback with ERR_INTERFACE_NOT_FOUND (or ERR_METHOD_NOT_FOUND) or just throw a corresponding RemoteError instance.

If an error occures, the method can pass any of the following to the callback:

  • a RemoteError instance;
  • an Error instance;
  • a number
  • a string.

See the Errors section for more info.

application.getMethods(interfaceName)

  • interfaceName {String}

Returns an array of all methods in a specified interface. If the interface does not exist, it should return any falsy value, such as null or undefined.

Interface: IServerApplicationsProvider

appsProvider.getApplication(applicationName)

  • applicationName {String} application name.
  • Return: {IApplication} application object.

Returns an application instance by name.
